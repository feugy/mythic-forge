En quelques mots

Mythic-Forge est un moteur de jeux en ligne pour créer des jeux 2D, multijoueurs et tour par tour.

C'est une explication limpide, non ? Non. Alors regardons de plus près :)

Vu de haut

C'est donc un moteur de jeu : vous voila dans la peau d'un Game Designer. Mythic-Forge est un outil qui vous permettra de créer un jeu, ce n'est pas un jeu en lui même.

Ensuite il est en ligne : le système se compose d'un serveur, et d'un client de jeu (une application Web), utilisable par vos joueurs sur un navigateur r(d)écent.

Survolez le diagramme pour en détailler les parties.

Hyperion: le serveur

Le serveur héberge les objets du jeu (cartes, personnages, évènements...) de même que l'ensemble des règles de jeu.

Ces règles indiquent ce que le joueur peux faire, et comment le système réagira en conséquence.

Le client de jeu

Il fourni aux joueurs tous les mécanismes de rendu et d'intéraction pour leur permettre de jouer.

Vous aurez la entièrement là main sur le client de jeu : à vous d'en définir l'identité visuelle, d'y expliquer les règles, d'y rajouter blog, forum et tout ce dont vous avez besoin pour faire grandir votre communauté.

Creusons un peu plus

Maintenant que vous avez saisi l'idée générale, voyons comment utilise Mythic-Forge.

Une autre application Web est fournie : c'est l'outil central pour gérer l'ensemble du système.

Avec cette application, vous manipulez tous les élements du server, ainsi que le code du client de jeu.

Le mot clé est "en direct". Pas besoin de recompiler quoi que ce soit. Inutile de redémarrer ou redéployer manuellement. Modifiez ou ajoutez quelque chose, et vous pouvez l'utiliser immédiatement, et tout est en ligne.

Rheia: le client d'administration

En tant que Game Developper/Designer, il vous permet de :

  • Créer et modéliser votre monde. Vous fixerez les caractéristiques des terrains, objets, personnages. Vous crérez vos cartes.
  • Rédiger les règles du jeu. Le moteur de jeu vous donne un contrôle total sur ce que le système fera.
  • Coder le client de jeu. En partant d'une base de code générique, vous personnaliserez votre client de jeu pour lui donner un comportement et un aspect unique et immersif.
  • Gérer le jeu en direct. Une fois en ligne, vous pourrez voir et modifier n'importe quel élément du monde en temps réel. Omniscience et ubiquité !

Détaillons un peu les téchnologies utilisées.

Dans Mythic-Forge, quasiement tout est en JavaScript. Ce langage offre quatre avantages significatifs:

  1. Portabilité : présent dans tous les navigateurs dignes de ce nom, il vous permet de joueur sur Pc, smart-phone, tablettes et même Tv !
  2. Flexibilité : interprété et faiblement typé, il offre la souplesse indispensable à un jeu qui évolue dans le temps.
  3. Sur le serveur et le client : un seul langage pour le règles coté serveur et le client de jeu. Moins d'effort d'apprentisage, et la possibilité de partager le code des deux cotés, sans pour autant se passer de sécurité.
  4. Outillé pour le rendu : les navigateurs web ont désormais les moyen de réaliser un rendu 2D de qualité, et ces outils sont utilisables nativement en JavaScript

En contrepartie, c'est un langage difficile a maitriser, et qui ne vous permettra pas (encore) de réaliser des jeu exploitant la 3D, ou exploitant toutes les capacités hardware du PC de vos joueurs.

Pour pallier de manière efficace aux défaut du JavaScript, vous utiliserez CoffeeScript: une syntaxe proche du Ruby dont l'interprétation génère un JavaScript optimisé et plus fiable.

Hyperion

Moteur : NodeJS, langage : CoffeeScript. NodeJS utilise V8 (l'interpreteur JavaScript des navigateurs Webkits), dont il a supprimé l'aspect DOM, et auquel il a rajouté la gestion des fichiers, sockets, processus...

Les données sont stockée dans une base de donnée MongoDB : performance et flexibilité.

Toutes les règles du jeu sont écrites en CoffeeScript.

La communication avec les clients de jeu utilise Socket.io : tout passe par les Websockets, un canal bi-directionnel permanent entre le navigateur et le serveur.

Rheia

Ecrit en CoffeeScript, et exploitant les dernières avancées du HTML5 et du CSS3. C'est une véritable application riche (Rich Internet Application).

Il utilise jQuery (manipulation du DOM) et jQuery-UI (bibliothèque de composants graphiques), RequireJS (gestion des dépendances entre les scripts) BackboneJS (structuration MVC du code).

Connecté en permanence grâce aux Socket.io, il exploite également Ace Editor pour éditer le code (les règles ainsi que les fichiers du client de jeu).

Le client de jeu

Comme Rheia, c'est une application riche en JavaScript/HTM5/CSS3.

Vous pourrez le coder en JavaScript, CoffeeScript, Css3 mais aussi Stylus.

Un kit de démarrage (jQuery, jQuery-UI, RequireJS, BackboneJS) propose les fonctionnalités de base du jeu (affichage de carte, exécutions des règles...). Mais vous pouvez utiliser toutes les librairies que vous souhaitez.

A vous ensuite de l'enrichir pour lui insuffler une personnalité et un Game Design unique. Vous aurrez besoin de le modifier en profondeur pour proposer à vos joueurs une véritable expérience de jeu !
